# CodeMan 部署清单 Skill

## 宿主环境说明

**CODEMAN_HOME** 路径规则：Cursor 为 `~/.cursor/skills/.codeman`，Claude Code / OpenCode 为 `~/.claude/skills/.codeman`，Trae 为 `~/.trae/skills/.codeman`。

---

## 概述

自动扫描代码变更，生成部署所需的完整信息清单，包括环境变量、数据库变更、依赖变更、部署步骤、风险提示等。

---

## 前置条件

执行前必须完成：

```
1. 确认测试全部通过
   → 读取 .codeman/docs/tests/test-report-latest.md
   → 确认所有 L1-L4 测试通过

2. 读取 .codeman/docs/design/INDEX.md
   → 了解架构变更记录

3. 读取 .codeman/docs/api/INDEX.md
   → 了解接口变更记录

4. 执行 Git diff 扫描
   → 与上一个发布 tag 对比
```

---

## 执行步骤

### Step 1: Git Diff 扫描

```bash
# 获取上一个发布 tag
LAST_TAG=$(git describe --tags --abbrev=0 2>/dev/null || echo "HEAD~1")

# 获取变更文件列表
git diff ${LAST_TAG}..HEAD --name-only

# 获取详细变更
git diff ${LAST_TAG}..HEAD --stat
```

### Step 2: 分析变更内容

**扫描以下变更类型：**

**① 环境变量变更**
- 扫描 `.env.example`、配置文件的变更
- 识别新增/修改/删除的环境变量

**② 数据库变更**
- 扫描 migration 文件或 schema 文件的变更
- 识别表结构变更（新增表/字段、修改字段、删除字段）
- 识别索引变更

**③ 依赖变更**
- 扫描 `package.json`、`go.mod`、`requirements.txt`、`Gemfile` 等的变更
- 识别新增/升级/移除的依赖包

**④ 配置变更**
- 扫描 nginx 配置、网关配置、Docker 配置的变更
- 识别第三方服务配置变更

**⑤ API 变更**
- 对比 api/INDEX.md 中的接口变更记录
- 识别新增/修改/删除的 API 端点

## 不确定即停

遇到以下情况必须暂停，严禁自行推测：

- 不确定环境变量是否需要配置（如新增的 API key、数据库连接串）
- 不确定数据库迁移脚本是否正确（特别是涉及数据丢失的操作）
- 不确定部署步骤的顺序（如先迁移数据库还是先部署代码）
- 不确定回滚方案是否可行
- 发现有未在部署清单中提及的依赖变更

**暂停格式：**
```
[需人工确认] 关于部署"{不确定点}"，我理解到以下几种可能：
1. {理解A}：[对应的部署方式/风险]
2. {理解B}：[对应的部署方式/风险]

请问您的意思是哪种？
```

### Step 7: 碎片文档归档建议

部署清单确认后，扫描碎片文档，识别可归档的过时文件：

**扫描规则：**
- `prd/feat-*.md`：检查是否有同一功能的多个版本（如 `feat-leads.md` 和 `feat-leads-v2.md`），旧版本建议归档
- `design/mod-*.md`：与实际代码对比（读代码目录结构），如果模块已被重写或删除，建议归档
- `api/api-*.md`：与实际路由文件对比，如果接口已不存在，建议归档

**输出归档建议（如有）：**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
碎片文档清理建议
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
以下碎片文档疑似过时：
  prd/feat-leads.md       — 已被 feat-leads-v2.md 替代
  design/mod-old-auth.md  — 对应代码已不存在

确认归档到 archive/？（回复"确认"批量归档，或指定只归档某些文件）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**用户确认后执行归档：**
1. 创建 `.codeman/docs/archive/` 目录（如不存在）
2. 将确认的碎片文件移入 `archive/`，保持原目录结构（如 `archive/prd/feat-leads.md`）
3. 更新对应的 INDEX.md，移除已归档条目
4. 不动 PRD-FULL.md 和 DESIGN-FULL.md（它们记录的是累积全貌，不受归档影响）

如无过时文件，跳过此步骤。
